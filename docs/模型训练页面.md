# 模型训练页面

## 目标

`模型训练`页面用于创建训练任务、查看训练任务列表、进入训练监控、查看训练日志、执行训练完成后的轻量操作。页面不负责管理数据集文件生命周期，也不负责管理 detection 默认推理模型、手动导出模型、评估模型或测试验证。

训练数据集来自 `数据集管理`页面维护的 OSS 数据集中心。创建远程训练任务时，前端使用数据集 `dataset_id` 调用后端远程训练接口。

## 当前页面结构

当前 `TrainingPage.vue` 主要包含：

- 顶部操作区：`新建训练任务`。
- 训练任务列表：
  - 任务 ID
  - 模型
  - 设备（后端返回的兼容展示字段，非新建任务输入项）
  - 进度
  - Epoch
  - 状态
  - 创建时间
  - 操作：监控、取消、删除
- 训练监控卡片：
  - 监控曲线
  - 运行日志
  - 下载运行日志
  - 手动刷新
  - 非终态任务轮询刷新
- 训练完成后的模型操作：
  - 导出训练报告
  - 下载权重
- 新建训练任务弹窗。

## 新建训练任务弹窗

当前表单字段：

- 训练数据集
- 检测场景
- 基础模型
- 训练轮数
- 批次大小
- 图像尺寸
- 优化器
- 初始学习率

当前创建任务接口：

```text
POST /api/training/remote/start
```

当前请求核心字段：

- `dataset_id`
- `model_name`
- `epochs`
- `img_size`
- `batch_size`
- `optimizer`
- `lr0`

当前训练任务创建不再传本地数据集路径，训练数据来自远程 OSS 数据集中心。

`训练设备`不再作为新建训练任务弹窗的用户输入项。训练资源选择由后端远程训练策略决定；`device` 字段目前仍可能出现在训练任务列表、监控摘要和训练报告中，用于展示旧数据或后端返回状态。

## 训练数据集选择设计

### 当前实现

训练数据集来自：

```text
GET /api/training/remote/datasets
```

前端只允许选择状态为 `UPLOADED` 的数据集：

```text
dataset.status === "UPLOADED"
```

数据集选项显示格式：

```text
数据集名字-上传时间-大小
```

上传时间优先级：

1. `server_verified_at`
2. `client_completed_at`
3. `updated_at`
4. `created_at`

当前选择框已支持按数据集名称筛选：

- `el-select filterable`
- 自定义筛选条件只匹配数据集名称
- 不匹配状态、大小、OSS 路径等字段

### 目标调整

训练页不再接受在本页面上传数据集，不再维护内嵌数据集上传弹窗或上传状态机。

目标交互：

1. `训练数据集`字段仍是单选框。
2. 单选框只展示 `UPLOADED` 数据集。
3. 单选框支持按名称搜索和筛选。
4. 在数据集选择区域旁提供 `上传数据集`按钮。
5. 点击 `上传数据集`时，在新页面打开：

```text
/datasets?openUpload=1
```

6. `数据集管理`页面读取 `openUpload=1` 后自动打开 `上传数据集`弹窗。
7. 上传完成后，用户回到训练页刷新数据集列表，或训练页在弹窗重新打开时重新拉取数据集列表。

设计约束：

- `TrainingPage.vue` 不应调用 `uploadDataset()`。
- `TrainingPage.vue` 不应包含上传数据集弹窗。
- `TrainingPage.vue` 不应保存 `uploadFile / uploadProgress / uploadDatasetName` 等数据集上传状态。
- 数据集上传的暂停、继续、取消、分片进度统一由 `DatasetPage.vue` 负责。

## 基础模型选择设计

### 当前实现

当前 `基础模型`字段是可筛选的下拉式单选框，选项包括：

- `YOLO11n (Nano · 最快)`
- `YOLO11s (Small · 轻量)`
- `YOLO11m (Medium · 均衡)`
- `YOLO11l (Large · 高精度)`
- `YOLO11x (X-Large · 最高精度)`

对应值：

- `yolo11n`
- `yolo11s`
- `yolo11m`
- `yolo11l`
- `yolo11x`

当前交互：

- 仍然是单选，不支持多选。
- 使用 `filterable` 的选择框。
- 支持按展示名搜索，例如 `Nano`、`Small`、`最快`。
- 支持按模型值搜索，例如 `yolo11n`、`yolo11m`。
- 选择后仍写入 `trainForm.model_name`。

第一阶段基础模型选项仍为固定列表；后续如果基础模型来自后端模型仓库，再改为远程搜索。

### 远程训练基础模型来源

远程训练不再让 Ultralytics 在容器内按 `yolo11n.pt` 等名称在线下载基础模型。基础模型统一预置在 OSS 中，并在启动 PAI-DLC 任务时作为独立 DataSource 挂载。

默认 OSS 目录：

```text
oss://{OSS_BUCKET}/base_model/
```

默认容器挂载目录：

```text
/mnt/base_model
```

模型选择与 OSS 文件映射：

```text
yolo11n -> /mnt/base_model/yolo11n.pt
yolo11s -> /mnt/base_model/yolo11s.pt
yolo11m -> /mnt/base_model/yolo11m.pt
yolo11l -> /mnt/base_model/yolo11l.pt
yolo11x -> /mnt/base_model/yolo11x.pt
```

相关环境变量：

```text
REMOTE_TRAIN_BASE_MODEL_PREFIX=base_model
PAI_BASE_MODEL_MOUNT_PATH=/mnt/base_model
```

启动远程训练时，DLC `DataSources` 包含三个挂载点：

- 数据集输入目录。
- 训练输出目录。
- 基础模型目录。

训练脚本会检查选择的基础模型文件是否存在；如果不存在，任务失败并在运行日志中提示需要先上传到 OSS `base_model` 目录。该策略避免容器每次训练都访问公网下载基础模型，也让基础模型版本由 OSS 目录显式管理。

## 检测场景设计

当前检测场景默认值为 `null`，页面显示：

```text
按数据集自动匹配/创建
```

该设计用于避免首次部署时强依赖固定场景 ID，也避免固定 `created_by=1` 之类数据库含义。

创建远程训练任务时，后端根据数据集关联的场景或数据集名解析场景。

## 训练监控设计

训练任务列表点击 `监控`后进入该任务监控卡片。

监控内容：

- 当前状态
- 当前模型
- 设备
- Epoch
- 指标卡片
- loss 曲线
- mAP / Precision / Recall 曲线
- 运行日志

刷新策略：

- 手动刷新按钮调用指标、状态和运行日志接口。
- 非终态任务按倒计时轮询；默认距离上次刷新 1 分钟后刷新。
- 用户主动刷新存在冷却期；距离上次刷新 20 秒内按钮置灰不可点击。
- 终态任务停止轮询。

运行日志说明：

- 监控页签固定展示 `运行日志`，不再只在失败时展示。
- PAI-DLC 任务运行时会把 OSS 训练输出前缀挂载到容器的 output 目录；容器在该挂载目录写入 `run.log`，对应 OSS 对象：

```text
train/jobs/{task_uuid}/run.log
```

- 前端通过后端接口读取 OSS 上的 `run.log`，不直接持有 OSS 长期凭据。
- 运行日志页面展示最新日志段，并显示后端返回的真实行号；行号对应完整 `run.log` 文件中的真实行号，而不是前端重新编号。
- `下载运行日志`由后端签发短期 OSS 下载 URL，下载完整 `run.log`。
- 如果 PAI-DLC 启动失败，容器没有运行，`run.log` 可能尚未生成；前端显示失败摘要和“运行日志尚未生成”，下载按钮不可用。
- `error_message`、`error_detail` 仍作为失败摘要和诊断详情字段保留，不再承担运行日志正文职责。

当前运行日志接口：

```text
GET /api/training/remote/logs/{task_id}
GET /api/training/remote/logs/{task_id}/download-url
```

为了配合 `模型管理`页面中的训练来源跳转，`TrainingPage.vue` 支持从 URL 参数恢复监控上下文：

```text
/training?task_uuid={task_uuid}
```

进入页面后自动选中对应训练任务，并显示该任务监控。当前实现使用 `task_uuid` 参数名，同时兼容该参数值为数值任务 ID 的情况。

## 删除训练记录

训练任务列表在终态任务上提供 `删除`操作。终态包括：

- `completed`
- `succeeded`
- `finished`
- `failed`
- `cancelled`
- `stopped`
- `aborted`

非终态任务（例如 `pending`、`running`）不能删除，必须先取消训练，避免 PAI-DLC 或本地训练进程继续写入训练产物。

删除训练记录的语义是级联删除该训练产生的模型与训练产物：

- 删除 `training_tasks` 记录。
- 删除该任务关联的 `training_metrics`。
- 删除该任务关联的 `remote_training_jobs`。
- 删除该任务产生的 `model_versions`。
- 删除模型和训练任务关联的 `model_artifact_locations`。
- 解除历史检测任务对待删除模型的引用。
- 解绑待删除模型关联的上传会话。
- 删除 OSS 训练输出目录 `remote_job.output_prefix` 下的对象，包括 `run.log`、`results.csv`、`weights/best.pt`、`_SUCCESS` 等。
- 删除本地训练输出目录 `TRAIN_OUTPUT_DIR/task_{task_uuid}`。

删除训练记录不会删除原始数据集。

如果该训练产生的模型仍是当前默认推理模型，删除操作必须拒绝，用户需要先到 `模型管理`页面切换默认模型，再删除训练记录。

当前删除接口：

```text
DELETE /api/training/tasks/{task_id}?cascade_models=true
```

前端删除确认文案必须明确提示：

```text
删除训练记录会同时删除该训练产生的模型、训练产物和运行日志，且不可恢复。
```

## 训练完成后的模型操作

训练任务完成后，训练页只保留与当前训练任务直接相关的轻量操作：

- `导出训练报告`：前端拉取训练指标和远程产物列表，导出 JSON 报告，文件名格式为 `training-report-{task_uuid_or_id}.json`。
- `下载权重`：下载当前训练任务产出的权重文件。

以下入口已从训练页移除：

- `评估模型`
- `导出模型`
- `测试验证`

移除原因：

- 训练完成后，系统会自动登记训练产物，包括训练报告、图表和模型权重，手动“导出模型”不再是必要流程。
- 模型生命周期管理、上传模型、删除模型、配置默认推理模型应由 `模型管理`页面承担。
- 模型评估和测试验证不再作为训练页底部操作入口提供；如果后续需要，应作为独立评估流程或模型管理能力重新设计。

## 与其他页面关系

### 与数据集管理页面

- `数据集管理`页面是数据集上传和删除的唯一入口。
- `模型训练`页面只选择已上传数据集。
- 训练页点击 `上传数据集`时，在新页面打开 `/datasets?openUpload=1`。

### 与模型管理页面

- `模型训练`页面不再展示“模型版本管理（全局默认模型切换）”。
- detection 默认推理模型设置功能由 `模型管理`页面承担。
- 训练页仅保留 `导出训练报告`和 `下载权重`；模型生命周期管理以 `模型管理`页面为准。
