# 远程冲突处理

## 2026-07-22 合并 `origin/main`

合并目标：把本地远程训练、数据集管理、内置场景初始化修复合并到远程最新 `main`，同时保留远程新增的 Multi-Agent、个人中心、RAG 知识库、登录页与侧边栏改动。

远程确实也修改了模型训练相关区域，需要人工处理冲突。

## 冲突文件

### `backend/app/agent/detection_agent.py`

远程改动：

- 引入 Day11 版 Agent 架构。
- 增加 Redis 对话记忆。
- 增加增强 SSE 事件：`thinking`、`tool_start`、`tool_end`、`text_chunk`、`done`。
- 工具拆分到 `app/agent/tools`。

本地改动：

- 异常不直接把底层错误返回给前端，详细信息只写日志。

处理结果：

- 保留远程 Agent 架构、工具事件、对话记忆和 `done` 事件。
- 保留本地安全错误处理：前端只收到统一失败文案，后端日志保留详细异常。

### `backend/app/training/training_service.py`

远程改动：

- 增加 `_resolve_weights_path`，支持从训练输出目录或 `ModelVersion.model_path` 回退查找模型权重。
- 调整模型评估、导出、下载流程以兼容已注册模型。

本地改动：

- 训练、评估、导出异常不把 Python 异常原文返回给前端。
- 对缺失权重、缺失 `results.csv` 等情况增加日志。
- 恢复 `data.yaml` 时避免二次异常覆盖主异常。

处理结果：

- 保留远程 `_resolve_weights_path` 回退逻辑。
- 保留本地安全错误返回和日志记录。
- `validate_model` 异常时回滚事务并返回统一错误文案。
- `data.yaml` 恢复失败只写日志，不覆盖原流程结果。

### `frontend/src/views/TrainingPage.vue`

远程改动：

- 训练页随远程 UI 改动一起更新。
- 检测场景选项指向固定 `scene_id = 3`。

本地改动：

- 训练任务不再依赖固定场景 ID，改为按数据集名自动匹配或创建场景。

处理结果：

- 保留远程页面结构。
- 检测场景选项选择本地方案：`scene_id = null`，显示为“按数据集自动匹配/创建”。
- 原因：数据库从 0 初始化时内置场景 ID 不稳定，固定 `3` 会重新引入部署风险。
