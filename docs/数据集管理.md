# 数据集管理

## 目标

`数据集管理`使用远程训练模块中已经实现的 OSS multipart 上传能力。OSS 是所有数据集文件的管理中心，后端只保存数据集元数据、上传会话、OSS object key、状态和训练关联关系，不再保存数据集 ZIP，也不再把数据集解压到 `backend/datasets` 目录作为权威数据源。

`数据集管理`页面也是训练数据集上传的唯一入口。其他页面如果需要上传数据集，应跳转到本页面并由本页面打开上传弹窗，不应在其他页面重复实现上传流程。

## 核心约束

- 数据集原始 ZIP 的权威位置是 OSS：`dataset_uploads.bucket + dataset_uploads.raw_object_key`。
- 原始 ZIP 的 Object Key 由后端生成，格式为 `{REMOTE_TRAIN_OSS_PREFIX}/{user_id}/{upload_id}/dataset.zip`；例如 `upload/1/upl_xxx/dataset.zip`。
- 后端不得持久化保存数据集 ZIP 文件。
- 后端不得以 `backend/datasets/{name}/yolo_dataset` 作为数据集管理页面的数据源。
- 浏览器不接触 AccessKey 或 Secret，只拿后端签发的短期 OSS multipart part URL。
- 上传完成不能只相信浏览器状态。可信状态由后端调用 OSS `CompleteMultipartUpload`，并通过 `HeadObject` 校验后推进。
- 数据集列表来自数据库 `dataset_uploads`，而不是扫描本地目录。
- 删除数据集时应删除或标记 OSS 中的数据集对象，并更新数据库状态；不应删除本地目录。
- OSS Bucket CORS 必须允许浏览器 `PUT` 分片请求，并暴露 `ETag` 响应头；前端需要 `ETag` 调用后端完成 multipart 合并。

## 上传流程

1. 前端选择单个 ZIP 文件，计算 `file_size`，使用服务端返回或默认的 `part_size` 计算总片数。
2. 前端调用 `POST /api/training/remote/uploads` 创建上传会话。数据集上传固定为 multipart，不再暴露上传模式选择。
3. 后端创建 `dataset_uploads` 记录，生成 OSS `raw_object_key`，调用 OSS `init_multipart_upload`，返回 `upload_id`、`dataset_id`、`part_size`、分片签名接口、完成接口和取消接口。
4. 前端按 part number 分批调用 `/multipart/parts/sign` 获取短期 PUT URL。
5. 前端逐片向 OSS 上传，每片完成后记录 `part_number` 与 OSS 返回的 `ETag`。
6. 前端展示上传进度：已上传片数/总片数、已上传 MB/总 MB、当前速度、预计剩余时间。
7. 所有分片上传完成后，前端调用 `/multipart/complete`，后端调用 OSS `CompleteMultipartUpload` 合并对象。
8. 后端 `CompleteMultipartUpload` 与 `HeadObject` 均成功后，上传会话直接进入 `UPLOADED`。
9. 远程训练使用 `dataset_id` 启动，输入数据来自 OSS raw ZIP 或处理后的 OSS prefix。

### 上传入口

`数据集管理`页面提供页面内上传入口：

- 顶部 `上传数据集` 按钮打开上传弹窗。
- 上传弹窗只接受单个 `.zip` 文件。
- 上传期间禁止关闭弹窗；用户需要先取消上传。
- 分片上传开始后，主按钮变为白底红边的 `取消上传`，左侧提供 `暂停上传 / 继续上传` 两态按钮。

其他页面需要上传数据集时，不再内嵌数据集上传弹窗。推荐跳转方式：

```text
/datasets?openUpload=1
```

前端行为：

1. 在新页面打开 `数据集管理`。
2. `DatasetPage.vue` 检测 `openUpload=1`。
3. 自动打开 `上传数据集`弹窗。
4. 上传完成后由用户回到训练页面刷新数据集列表或重新选择数据集。

### 上传控件交互

上传控件应保持单文件体验：

- 未选择文件时显示拖拽上传栏。
- 用户拖入或选择文件后，拖拽上传栏替换为单个文件名展示块。
- 文件名展示块显示文件本身名称，例如 `chest_xray_v2.zip`。
- 不显示本地路径，不显示 OSS 路径。
- 文件名过长时截断显示。
- 鼠标悬停在文件名上时显示完整文件名。
- 删除已选文件后恢复拖拽上传栏。

## 数据集列表

`数据集管理`页面展示 `dataset_uploads` 中当前用户可见的数据集记录。至少包含：

- 数据集名称：`dataset_name`
- 数据集 ID：`dataset_uuid`
- 上传 ID：`upload_uuid`
- 状态：`INITIATED / UPLOADING / UPLOADED / FAILED / EXPIRED / CANCELLED`
- 文件大小：`expected_size / actual_size`
- OSS 位置：`bucket + raw_object_key`
- 上传时间、完成时间、失败原因

本地图片数量统计不再从本地目录读取。后续如果需要训练集/验证集图片数量，应由预处理任务写入 `manifest.json`，后端从 OSS manifest 或数据库快照展示。

## 前后端 API 整理

保留并使用远程训练模块 API：

- `GET /api/training/remote/datasets`：查询 OSS 数据集中心。
- `DELETE /api/training/remote/datasets/{dataset_ref}`：删除或取消一个 OSS 数据集。
- `POST /api/training/remote/uploads`：创建 OSS multipart 上传会话。
- `POST /api/training/remote/uploads/{upload_id}/multipart/parts/sign`：签发一批分片 PUT URL。
- `POST /api/training/remote/uploads/{upload_id}/multipart/complete`：后端调用 OSS 完成 multipart 合并。
- `POST /api/training/remote/uploads/{upload_id}/multipart/abort`：取消未完成的 multipart 上传。

旧接口处理：

- `/api/training/datasets`
- `/api/training/datasets/upload`
- `/api/training/datasets/{...}`

这些接口属于本地数据集时代的冗余接口。整理 API 时应消除，避免继续出现“本地数据集目录”和“OSS 数据集中心”两套语义。

  - `POST /api/training/remote/uploads/{upload_id}/multipart/abort`

- 旧的本地数据集 API 已从 `backend/app/api/training.py` 移除。
- 数据集列表已改为查询 `dataset_uploads`，不再扫描 `backend/datasets`。
- 删除数据集会取消未完成 multipart、删除已知 OSS 对象，并把数据库记录标记为 `CANCELLED`；管理列表默认不展示已取消记录。
- 数据集上传已取消 `CLIENT_COMPLETED`、`READY` 和 OSS/FC 完成事件回调依赖；后端完成 multipart 合并并通过 `HeadObject` 后直接进入 `UPLOADED`。

## 与模型训练页面的关系

`模型训练`页面的新建训练任务弹窗只负责选择训练数据集，不负责上传数据集。

目标交互：

1. 训练页数据集选择框只展示状态为 `UPLOADED` 的数据集。
2. 训练页提供 `上传数据集`跳转按钮。
3. 点击该按钮时，在新页面打开 `/datasets?openUpload=1`。
4. 数据集上传完成后，训练页通过刷新数据集列表获取新数据集。

这样可以避免训练页和数据集管理页维护两套上传状态机。
