## 一、项目背景与目标

### 1.1 项目背景

胸部 X 光（Chest X-Ray）是全球最常用的医学影像检查手段。放射科医生每天需要阅读大量胸片，工作强度大。本系统旨在构建一个**以对话为核心交互方式的多智能体 AI 辅助诊断平台**，让医生像与资深同事讨论病例一样，通过自然语言完成影像上传、检测分析、报告生成等全流程操作。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **对话驱动** | 主要操作通过自然语言对话完成，降低学习成本 |
| **多 Agent 协作** | 由 Supervisor Agent 自动路由到检测/分析/报告/问答/训练等专业 Agent |
| **病灶检测** | 自动识别并框出 10 种常见胸部病变 |
| **智能报告** | 检测结果自动生成结构化医学报告 |
| **持续学习** | 支持新数据标注、模型微调、版本管理 |

## 二、核心设计理念：对话即界面

### 2.1 设计哲学

```
传统系统：用户学习功能菜单 → 找到对应页面 → 点击按钮 → 等待结果
本系统：  用户说一句话 → AI 理解意图 → 自动调度 Agent → 返回结果
```

### 2.2 交互原则

| 原则 | 说明 |
|------|------|
| **对话优先** | 所有功能均可通过对话触发，图形界面为辅助手段 |
| **意图理解** | Supervisor Agent 自动理解用户意图并路由到合适的专业 Agent |
| **上下文保持** | 对话中保持患者/影像上下文，支持多轮追问 |
| **渐进展示** | 复杂结果（如检测框）以卡片/图片形式嵌入对话流 |
| **降级兼容** | 对话无法覆盖的功能（如数据看板），通过侧边导航完成 |

### 2.3 典型对话场景

```
用户："帮我分析一下这张胸片"
  → Supervisor 识别意图 → 路由到 Detection Agent
  → Agent 引导上传图片 → 调用 YOLO 模型 → 返回检测结果卡片

用户："这个结节严重吗？"
  → Supervisor 保持上下文（知道刚才检测了哪张图）
  → 路由到 Analysis Agent → 分析结节大小/位置/形态 → 给出建议

用户："帮我生成一份报告"
  → Supervisor → Report Agent → 结合检测结果 + 分析意见 → 输出结构化报告
```

### 3.1 Agent 角色定义

```

│                    多 Agent 协作架构 (LangGraph)                  │
│                                                                  │
│                        ┌─────────────────┐                      │
│                        │  Supervisor      │                      │
│                        │  Agent (调度者)   │                      │
│                        │  意图识别+路由    │                      │
│                        └────────┬────────┘                      │
│                                 │                                │
│           ┌─────────┬──────────┼──────────┬─────────┐           │
│           ▼         ▼          ▼          ▼         ▼           │
│     ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐     │
│     │Detection│ │Analysis│ │ Report │ │  QA    │ │Training│     │
│     │ Agent  │ │ Agent  │ │ Agent  │ │ Agent  │ │ Agent  │     │
│     │ 检测    │ │ 分析    │ │ 报告    │ │ 问答    │ │ 训练    │     │
│     └────────┘ └────────┘ └────────┘ └────────┘ └────────┘     │
│          │           │          │          │          │          │
│          ▼           ▼          ▼          ▼          ▼          │
│     ┌──────────────────────────────────────────────────┐        │
│     │              共享工具层 (Tools)                    │        │
│     │  YOLO推理 | 图像处理 | 数据库查询 | MinIO存取 |    │        │
│     │  知识检索 | 统计计算 | 格式转换   | 报告导出  |    │        │
│     └──────────────────────────────────────────────────┘        │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 各 Agent 职责详表

| Agent | 中文名 | 触发场景 | 核心能力 | 依赖工具 |
|-------|--------|----------|----------|----------|
| **Supervisor Agent** | 调度智能体 | 所有对话入口 | 意图识别、上下文管理、Agent 路由、多步骤任务编排 | — |
| **Detection Agent** | 检测智能体 | "帮我分析这张胸片" | 引导上传、调用 YOLOv11、返回带框检测结果 | YOLO 推理、MinIO 上传 |
| **Analysis Agent** | 分析智能体 | "这个结节怎么看？" | 解读检测结果、评估风险等级、鉴别诊断建议 | 检测结果查询、知识库 |
| **Report Agent** | 报告智能体 | "生成一份报告" | 结构化报告生成、PDF 导出、历史报告对比 | 模板引擎、PDF 生成器 |
| **QA Agent** | 问答智能体 | "什么是肺实变？" | 医学知识问答、影像征象解释、指南查询 | RAG 知识库 |
| **Training Agent** | 训练智能体 | "用新数据微调模型" | 数据集上传、格式转换、启动训练、监控进度 | 训练脚本、数据转换器 |

### 3.3 Agent 协作示例：完整诊断流程

```
用户："请帮我全面分析患者 P001 的胸片"

Supervisor Agent:
  ├── ① 识别意图：完整诊断流程
  ├── ② 检查上下文：P001 是否存在？有无已上传影像？
  └── ③ 路由 → Detection Agent

Detection Agent:
  ├── ④ 调用 YOLOv11 模型推理
  └── ⑤ 返回：发现 3 个病灶（2个结节 + 1个实变），生成标注图片

Supervisor Agent → 路由 → Analysis Agent

Analysis Agent:
  ├── ⑥ 分析每个病灶：大小、位置、形态特征
  ├── ⑦ 风险评级：结节1（中风险，建议随访）、实变（中风险，疑似感染）
  └── ⑧ 生成鉴别诊断列表

Supervisor Agent → 询问用户："需要生成正式报告吗？"

用户："需要"

Report Agent:
  ├── ⑨ 整合检测结果 + 分析意见
  └── ⑩ 输出报告预览 + PDF 下载链接
```

|--------|------|----------|-----------|

| **P0** | 检测结果解读分析 | ✅ Analysis Agent | — |
| **P0** | 医学报告生成 | ✅ "生成报告" | — |
| **P1** | 历史记录查询 | ✅ "查看 P001 历史" | — |
| **P1** | 医学知识问答 | ✅ QA Agent | — |
| **P1** | 数据统计看板 | ❌ 图表不便对话 | `/dashboard` |
| **P2** | 模型训练管理 | ✅ Training Agent | `/training` |
| **P2** | 用户/角色管理 | ❌ 管理后台 | 管理后台 |

### 5.2 F1 — 对话式影像检测（核心 P0）

| 属性 | 内容 |
|------|------|
| **触发方式** | 对话中输入"帮我分析这张胸片"并上传图片 |
| **Agent 流程** | Supervisor → Detection Agent → 返回结果卡片 |
| **检测类别** | Nodule, Mass, Consolidation, Effusion, Atelectasis, Pneumothorax, Emphysema, Fibrosis, Calcification, Fracture |
| **输出** | 原图叠加检测框 + 病灶列表（类别/置信度/位置/尺寸） |

### 5.3 F2 — 智能分析解读（核心 P0）

| 属性 | 内容 |
|------|------|
| **触发方式** | 检测后自动触发或用户追问 |
| **Agent** | Analysis Agent |
| **分析维度** | 病灶特征描述、风险评级（low/medium/high/critical）、鉴别诊断、随访建议 |

### 5.4 F3 — 结构化报告生成（核心 P0）

| 属性 | 内容 |
|------|------|
| **触发方式** | "生成报告" / "出报告" |
| **Agent** | Report Agent |
| **输出格式** | 对话中预览卡片 + PDF 下载 |

### 5.5 F4 — 医学知识问答（P1）

| 属性 | 内容 |
|------|------|
| **触发方式** | 自然语言提问 |
| **Agent** | QA Agent（基于 RAG 检索医学知识库） |

### 5.6 F5 — 历史记录查询（P1）

| 属性 | 内容 |
|------|------|
| **对话方式** | "查看 P001 的检测历史""对比两次检测结果" |
| **功能** | 查看详情、对比变化、历史报告查看 |

### 5.7 F6 — 数据统计看板（P1，仅侧边栏）

| 属性 | 内容 |
|------|------|
| **访问方式** | 侧边栏导航 → `/dashboard` |
| **统计维度** | 检测总量趋势、各类病灶检出率、风险等级分布 |

### 5.8 F7 — 模型训练管理（P2）

| 属性 | 内容 |

## 八、非功能需求

### 8.1 性能要求

| 指标 | 要求 |
|------|------|
| 单张胸片检测时间 | ≤ 3 秒（GPU）/ ≤ 10 秒（CPU） |
| Agent 对话响应 | 首 Token ≤ 2 秒 |
| SSE 流式输出 | 逐 Token 展示，无卡顿 |
| 并发对话 | ≥ 20 用户 |

### 8.2 安全要求

| 要求 | 说明 |
|------|------|
| 传输加密 | HTTPS |
| 认证 | JWT Token（已有） |
| 患者隐私 | 患者编号脱敏，不存储真实身份信息 |
| 操作审计 | 所有关键操作记录到 `operation_logs` |

### 8.3 可用性要求

| 要求 | 说明 |
|------|------|
| 界面语言 | 中文 |
| 响应式 | 支持 1366×768 及以上 |
| 浏览器兼容 | Chrome / Edge / Firefox 最新版 |

## 十、技术约束与依赖
