# PPT 生成提示词 — ChestVision 胸片X光智能分析系统（前端）

请根据以下项目信息，制作一份专业的前端技术展示 PPT，面向评委/导师/投资人，风格为**医疗科技 + 现代简约**。

---

## 一、PPT 整体要求

- **页数**：14-15 页
- **配色**：主色 #2A9D8F（青绿），辅色 #1a1f2e（深蓝黑），背景 #F0F4F8（浅灰）
- **字体**：标题用粗体，正文用无衬线字体（推荐 HarmonyOS Sans / Inter）
- **风格**：医疗科技风，卡片式排版，大圆角，弥散阴影，克制留白
- **图标**：使用简洁线性图标或 emoji（🫁🧠🔬📊📋💬📁⚙️）
- **动画**：淡入 + 上移

---

## 二、PPT 结构建议

### 第1页：封面
- 标题：**ChestVision — 胸片X光智能分析系统**
- 副标题：**前端技术方案与实现**
- 底部：项目名称 + 日期
- 背景：青绿渐变 + 肺部标识

### 第2页：项目概述
- 一句话定位：基于深度学习（YOLOv11）的胸部X光 AI 辅助诊断平台
- 三大核心能力：
  - 🧠 AI 病灶检测（10 种胸部病变，mAP>90%）
  - 💬 多 Agent 智能对话（Supervisor + Detection + Diagnosis + Report）
  - 📊 全流程数据管理（患者 → 检测 → 病例 → 报告）
- 技术栈标签云：Vue 3 / Vite / Element Plus / ECharts / Pinia / SSE

### 第3页：系统架构
- 前端架构图（三层）：
  - 展示层：Vue 3 + Element Plus（10个页面）
  - 数据层：Pinia（userStore + agentStore）+ Axios 拦截器
  - 通信层：REST API + SSE 流式
- 后端简述：FastAPI + LangGraph + PostgreSQL + Redis + MinIO

### 第4页：用户角色与权限设计
- 三角色矩阵图：
  - 🏥 管理员：全部功能 + 模型训练 + 分配医生
  - 👨‍⚕️ 医生：AI 辅助诊断 + 患者管理 + 病例编辑
  - 🧑 患者：上传胸片 + 查看报告 + 选择推荐医生
- 路由守卫 + JWT Token 鉴权机制

### 第5-6页：核心功能 — 登录与注册（重点）
- **登录面板**：用户名 + 密码 + "记住我" + "忘记密码？"
- **注册面板**：姓名 + 邮箱 + 邮箱验证码（60s倒计时）+ 用户类型选择 + 密码
- **双面板滑动切换**：登录 ↔ 注册，中间品牌色 overlay
- **视觉设计**：毛玻璃卡片 + 背景渐变光晕 + 左上角校徽 logo
- **品牌展示区**：🫁 logo + "ChestVision" + 三大产品亮点文案
- **技术点**：JWT 认证、邮箱验证码防刷、角色选择分发

### 第7-8页：核心功能 — 智能对话（重点）
- **欢迎模式**（无对话时）：
  - 居中 🫁 logo（呼吸动画）+ "你好，{用户名}" + "开启智能医疗问答"
  - 3 个快捷提问卡片 + 快捷检测按钮（单图/批量）
  - 患者选择器（医生可见）+ 文件上传 + 文本输入
  - 侧边栏可展开查看历史对话
- **对话模式**（有消息后）：
  - 消息气泡：用户青色渐变（右）+ AI 白底圆角（左，带 🫁 头像）
  - Multi-Agent 6 节点流程可视化（Supervisor → Detection → Diagnosis → Report → QA → Summarize）
  - 工具调用状态展示 + 知识库来源标注
  - 检测结果卡片内嵌 + PDF 报告一键下载
- **技术亮点**：
  - SSE 流式响应（Fetch + ReadableStream + AbortController）
  - Markdown 实时渲染 + 打字动画
  - 快捷检测模式（跳过对话，直接调用 YOLO 引擎）

### 第9页：核心功能 — 数据看板
- KPI 统计卡片（4 个指标）：检测总次数 / 病灶总数 / 平均耗时 / 高风险案例
- ECharts 图表：检测趋势折线图 + 病灶类型饼图 + 风险等级分布
- 医生工作量统计表（管理者视角）

### 第10页：核心功能 — 患者与病例管理
- 患者管理：列表 + 医生分配 + AI 推荐审核
- 病例管理：CRUD + 附件上传 + 打印
- 个人中心：健康档案 + 密码修改

### 第11页：核心功能 — 模型训练
- 训练任务管理：进度条 + 实时指标监控（损失曲线 + mAP 曲线）
- 模型版本管理：默认模型切换 + 验证评估 + 导出/下载权重

### 第12页：技术亮点
- **SSE 流式交互**：低延迟实时 AI 回复
- **Multi-Agent 可视化**：6 节点协作流程透明化
- **全局错误监控**：三层错误捕获 + 本地日志
- **响应式设计**：侧边栏可折叠（250px ↔ 64px）
- **主题定制**：基于 SCSS 变量的医疗色系统一覆盖 Element Plus
- **邮箱验证码**：防刷注册 + 60s 倒计时

### 第13页：设计系统
- 色板展示：主色 #2A9D8F + 辅色 #1a1f2e + 功能色
- 组件展示：按钮 / 卡片 / 表格 / 标签 / 输入框
- 设计原则：医疗专业感 + 温暖人性化 + 信息层次清晰

### 第14页：项目总结与展望
- 已完成：10 个功能页、3 种用户角色、SSE 流式对话、Multi-Agent 协作
- 待优化：UI 进一步品牌化、移动端适配、E2E 测试覆盖
- 技术价值：将 AI 影像诊断能力产品化，降低医生使用门槛
